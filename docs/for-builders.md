# For Builders — Workflow-as-Code

You build AI workflows in the Noxus UI, then pull them into version-
controlled Python files. Your team reviews changes in pull requests.
Non-technical colleagues use `make chat` to interact with your agents.

## The loop

    UI sketch  →  make pull ID=<id>  →  edit in VS Code  →  PR  →  make push

## Setup

    git clone <your-team-repo-url>
    cd <repo-name>
    make setup          # .venv + deps + hooks
    $EDITOR .env        # set NOXUS_API_KEY

## Daily workflow

### 1. Build in the UI

Create your workflow visually at noxus.ai. Configure nodes, connect
edges, test it interactively.

### 2. Pull to code

    make pull ID=<workflow-id>

This generates `examples/NN_<slug>.py` — a self-contained Python file
that recreates the workflow. The file carries a provenance header:

```python
# generated by noxuslab pull from <id> @ 2026-05-08T12:00:00Z
```

### 3. Edit and iterate

Open the file in your IDE. Refactor, add logic, parameterize. The
generated code uses the `noxus-sdk` directly — no magic.

### 4. Review in a PR

    git switch -c feat/improve-onboarding
    git add examples/NN_onboarding.py
    git commit -m "feat(workflow): improve onboarding prompts"
    git push -u origin HEAD

CI runs lint + tests automatically.

### 5. Push back to Noxus

    make push FILE=examples/NN_onboarding.py

The workflow is now live in the UI again, with your code changes applied.

## Configure agents for your team

1. Build an agent in the Noxus UI (model, persona, tools, KB access)
2. Share the agent ID with your team: `make chat AGENT=<id>`
3. Non-technical users can now talk to your agent from terminal

## Discovery

    make list           # show all workflows
    make agents         # show all agents

## Scaffold a new project

    make new NAME=client-onboarding

Copies examples, Makefile, and bin/ scripts into a fresh directory.
Push to a new GitHub repo as a standalone project.

## Architecture

```
examples/       one-file scripts, each does one thing
noxuslab/       CLI + codegen (pull/push/chat/ask)
bin/            POSIX scripts used by both Makefile and CI
tests/          pytest suite (offline, mocked)
docs/           you are here
```

## Key commands

| Command | Purpose |
|---------|---------|
| `make hello` | Build a 3-node workflow (sanity check) |
| `make run ID=<id>` | Execute a workflow |
| `make pull ID=<id>` | Workflow → Python file |
| `make push FILE=<f>` | Python file → Workflow |
| `make diff ID=<id> FILE=<f>` | Show what `push` would change |
| `make chat AGENT=<id>` | Interactive agent chat |
| `make ask Q="..." AGENT=<id>` | One-shot question (pipe-friendly) |
| `make agents` | List agents in the workspace |
| `make lint` | Check code style |
| `make ci` | Full local CI (lint + test) |
